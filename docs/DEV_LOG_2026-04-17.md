## 三、今天完成的工作

### 3.1 主题系统重构（白屏+三主题失灵）

**问题：** 三主题（日/夜/护眼）切换后颜色不变，选中态背景盖住文字（尤其是 light/eye 主题）。

**根因：**
1. `useTheme.ts` 里的 CSS 变量名是 `--theme-bg`/`--theme-text`，但 `base.css` 用的是 `--bg`/`--text`，完全对不上
2. App.vue 的 data-theme 绑定存在但 CSS 变量没对应上
3. 各组件选中态（`.active`、`.selected`）用了硬编码颜色，light/eye 主题下看不清

**修复：**
- 重写 `useTheme.ts`，统一用 `--bg`/`--text`/`--surface`/`--border`/`--gold`/`--selected-bg`/`--selected-text`
- `base.css` 的 `:root` 定义 dark 主题变量，`[data-theme="light"]` 和 `[data-theme="eye"]` 各有一套
- 所有 `.active` / `.selected` 样式统一改成 `var(--selected-bg)` 和 `var(--selected-text)`
- App.vue Header 加了三主题切换按钮（🌙☀️🔅循环）

**文件：** `src/composables/useTheme.ts`、`src/assets/styles/base.css`、`src/App.vue`

### 3.8 release build 白屏（最关键 bug）

**问题：** dev 模式（`npm run tauri dev`）完全正常，但 release build 白屏，连 Header 都消失了。

**根因：**
- `tauri.conf.json` 的 `bundle.resources` 配置：
  ```json
  "resources": {
    "../dist": "dist"   // ← 错误！把前端放到了 Resources/dist/
  }
  ```
- Tauri v2 WebView 默认加载 `tauri://localhost/index.html` → 对应 `Resources/index.html`（根目录）
- 实际文件在 `Resources/dist/index.html`，路径不匹配，WebView 拿到空内容 → 白屏

**修复：**
```json
"resources": {
  "../dist": ""   // ← 空字符串！把 dist 内容放到 resources 根目录
}
```

**关键经验：** Tauri v2 的 `bundle.resources` 中 value 为空字符串 `""` 表示"放到根目录"，为 `"dist"` 表示放到 `dist/` 子目录。

**文件：** `src-tauri/tauri.conf.json`

## 四、Vite 配置关键点

**文件：** `vite.config.ts`

```typescript
export default defineConfig({
  plugins: [vue()],
  base: './',        // ← 关键！相对路径，打包后资源用 ./assets/...
  clearScreen: false,
  server: {
    port: 5173,
    strictPort: true,
  },
  build: {
    target: process.env.TAURI_PLATFORM === 'windows' ? 'chrome105' : 'safari13',
    minify: !process.env.TAURI_DEBUG ? 'esbuild' : false,
    sourcemap: !!process.env.TAURI_DEBUG,
  },
})
```

**`base: './'` 是必须加的**——没有这行，打包后 HTML 里资源路径是 `/assets/index-BjhTkKxZ.js`（绝对路径），在 App 里没有 Web 服务器，无法访问。加上后变成 `./assets/index-BjhTkKxZ.js`（相对路径），可以正常解析。

## 八、经验总结

1. **Tauri v2 bundle.resources 的 value 不能想当然**：`"dist"` 不等于"放到根目录"，空字符串 `""` 才是根目录
2. **Vite 打包后资源路径**：`base: './'` 是必须的，否则 App 里无法加载资源
3. **Vue 模板里慎用 `??`**：在 `v-for` 的 `in` 表达式里用 `??` 会导致 TS 编译报错，用 `computed` 包装一下
4. **dev 模式 vs release 行为差异**：dev 模式走 localhost，有问题容易忽略；release 才是真实环境
5. **Tauri dev 模式内存消耗大**：Mac mini 只有 16GB 内存，dev 模式长时间跑容易被 OOM killer
